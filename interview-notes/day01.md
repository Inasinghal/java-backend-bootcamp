# Day 1 – Interview Questions & Answers

## 1. What is Java?

Java is a high-level, object-oriented, class-based programming language developed by Sun Microsystems (now Oracle). It is designed to be platform-independent, meaning a Java program can run on any operating system that has a Java Virtual Machine (JVM).

Java is widely used for backend development because it is secure, reliable, scalable, and has a rich ecosystem of frameworks such as Spring Boot.

**Interview Tip:**
Avoid saying "Java is only an object-oriented language." Java also supports functional programming features like lambda expressions and streams.

---

## 2. Why is Java platform-independent?

Java is platform-independent because the Java compiler converts source code into bytecode instead of machine code. This bytecode is executed by the Java Virtual Machine (JVM), and each operating system has its own JVM implementation.

As a result, the same compiled Java program can run on Windows, Linux, or macOS without changing the code.

**Flow:**

Java Source Code (.java)

↓

Compiler (javac)

↓

Bytecode (.class)

↓

JVM

↓

Machine Code

↓

CPU

This concept is known as **"Write Once, Run Anywhere (WORA)."**

---

## 3. Difference between JDK, JRE, and JVM

### JVM (Java Virtual Machine)

The JVM is responsible for running Java bytecode. It manages memory, performs garbage collection, loads classes, executes bytecode, and converts it into machine code.

### JRE (Java Runtime Environment)

The JRE contains the JVM and the required libraries needed to run Java applications. It cannot compile Java code.

### JDK (Java Development Kit)

The JDK contains everything needed for Java development, including the JRE, JVM, Java compiler (`javac`), debugger, and other development tools.

**Simple Analogy**

* JVM = Engine
* JRE = Engine + Fuel
* JDK = Complete Car Factory

As developers, we install the JDK because it already contains the JRE and JVM.

---

## 4. What is Bytecode?

Bytecode is the intermediate code generated by the Java compiler after compiling a `.java` file.

It is not machine code and cannot run directly on the CPU. Instead, the JVM reads the bytecode, verifies it, and converts it into machine instructions that the operating system understands.

This intermediate representation is what makes Java platform-independent.

---

## 5. What happens when a Java program runs?

When a Java program runs, the following steps occur:

1. The developer writes Java source code (`.java`).
2. The Java compiler (`javac`) compiles the source code into bytecode (`.class`).
3. The JVM loads the class file using the Class Loader.
4. The JVM verifies the bytecode for security and correctness.
5. Memory is allocated for the program.
6. The JVM executes the `main()` method.
7. Frequently used bytecode is converted into native machine code using the Just-In-Time (JIT) compiler for better performance.
8. The CPU executes the machine code.
9. The Garbage Collector automatically frees unused memory.

This execution flow is a favorite interview topic because it tests understanding of how Java works internally.

---

## 6. Why is `main()` static?

The `main()` method is static so that the JVM can call it without creating an object of the class.

When a Java program starts, no objects exist yet. If `main()` were non-static, the JVM would first need to create an object before it could start the program, creating a circular dependency.

Making `main()` static allows the JVM to invoke it directly as the application's entry point.

## 7. Primitive vs Reference Types

### Primitive Types

Primitive types store actual values directly in memory.

Examples:

* byte
* short
* int
* long
* float
* double
* char
* boolean

Example:

```java
int age = 30;
```

Here, the value `30` is stored directly.

### Reference Types

Reference types store the memory address (reference) of an object rather than the object itself.

Examples:

* String
* Array
* List
* Custom Classes

Example:

```java
String name = "Ayush";
```

The variable `name` stores a reference to a `String` object.

**Key Difference**

Primitive variables hold values.

Reference variables hold addresses pointing to objects.

## 8. What is Type Casting?

Type casting is the process of converting one data type into another.

### Implicit Casting (Widening)

Performed automatically when converting a smaller type into a larger compatible type.

```java
int number = 10;
double value = number;
```

Output:

```
10.0
```

### Explicit Casting (Narrowing)

Performed manually when converting a larger type into a smaller type.

```java
double salary = 65000.75;
int amount = (int) salary;
```

Output:

```
65000
```

The decimal part is lost because narrowing conversion may lose data.

## 9. Difference between `float` and `double`

| Feature     | float                | double                         |
| ----------- | -------------------- | ------------------------------ |
| Size        | 4 bytes              | 8 bytes                        |
| Precision   | ~6–7 decimal digits  | ~15–16 decimal digits          |
| Suffix      | `f`                  | No suffix needed               |
| Performance | Slightly less memory | More precision (commonly used) |

Example:

```java
float pi = 3.14f;
double value = 3.141592653589793;
```

In most Java applications, `double` is preferred because of its higher precision.

## 10. Why is Java suitable for backend development?

Java is widely used for backend development because it offers:

* Platform independence
* High performance through JVM optimizations and the JIT compiler
* Strong object-oriented design
* Automatic memory management with Garbage Collection
* Excellent multithreading support
* Robust security features
* Mature frameworks such as Spring Boot
* Rich ecosystem of libraries and tools
* Scalability for enterprise applications
* Strong community support and long-term stability

Large organizations use Java to build banking systems, e-commerce platforms, telecom software, healthcare applications, and cloud services because it is reliable, maintainable, and performs well under heavy workloads.

# Quick Revision

* Java is compiled into **bytecode**, not machine code.
* The **JVM** executes bytecode.
* The **JRE** provides the runtime environment.
* The **JDK** provides development tools and includes both the JRE and JVM.
* Java is platform-independent because of the JVM.
* Primitive types store values directly; reference types store object references.
* `main()` is static so the JVM can start the program without creating an object.
* `double` is generally preferred over `float` because it provides greater precision.
